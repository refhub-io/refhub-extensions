// refhub_ext

extension_settings

dark-first capture for current tab metadata

// connection

required

this extension uses a refhub API key for vault listing and item creation. create the key in /profile-edit under api_keys.

required_scopes vaults:read • vaults:write
optional_scope vault restriction only if you want least-privilege keys
refhub_api_base_url -
refhub_app_url -

release builds keep refhub endpoints fixed. local or self-hosted targets are available through a dev override build.

// setup_flow

  1. sign in to refhub web app.
  2. open /profile-edit and switch to api_keys.
  3. create a key with vaults:read and vaults:write.
  4. paste the key here. release builds already point at production refhub.

// what_is_sent_on_save

  • normalized page metadata such as title, authors, DOI, year, journal, abstract, and URL
  • the selected refhub vault ID
  • no full page HTML is sent by this prototype

// extension_permissions

  • activeTab so capture only runs on the page you explicitly opened from.
  • scripting so the extension can read metadata from that active tab.
  • storage so API key and last vault selection persist locally, with URL overrides only in dev builds.
  • host access is limited to the bundled refhub API origin in release builds.
